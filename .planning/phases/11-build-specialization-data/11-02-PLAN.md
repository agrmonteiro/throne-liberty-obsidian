```python
def _try_export_specialization(page) -> dict | None:
    """
    Tenta capturar o JSON de especialização via o botão 'Export to Share'
    no Build Editor do Questlog. Retorna None se falhar — fallback gracioso.

    NOTA: Seletores confirmados via inspeção DOM (veja checkpoint Task 1).
    Se o botão não for encontrado, o scraper retorna normalmente sem maestrias.
    """
    try:
        # Usar o seletor confirmado no checkpoint. Fallback: múltiplos seletores.
        # Substituir pelos seletores exatos identificados no checkpoint se disponíveis.
        export_btn = page.query_selector(
            '[aria-label="Export to Share"], '
            '[title="Export to Share"], '
            'button[data-testid="export-button"], '
            'button[class*="export"]'
        )
        if not export_btn:
            print("[scraper] Botão de export não encontrado — sem maestrias", flush=True)
            return None

        print("[scraper] Botão de export encontrado — clicando...", flush=True)
        export_btn.click()

        # Aguardar modal ou elemento com o JSON aparecer
        page.wait_for_timeout(1500)

        # Estratégia 1: ler de textarea/pre no modal (mais comum)
        json_el = page.query_selector(
            'textarea[readonly], '
            'pre[data-export-json], '
            '.export-modal textarea, '
            'dialog textarea, '
            '[role="dialog"] textarea'
        )
        if json_el:
            try:
                raw_json = json_el.input_value() or json_el.inner_text()
                if raw_json and raw_json.strip().startswith('{'):
                    parsed = json.loads(raw_json.strip())
                    print("[scraper] JSON de export capturado com sucesso", flush=True)
                    return {
                        "mainHand":       parsed.get("mainHand"),
                        "offHand":        parsed.get("offHand"),
                        "specialization": parsed.get("specialization") or [],
                    }
            except (json.JSONDecodeError, Exception) as e:
                print(f"[scraper] Falha ao parsear JSON do modal: {e}", flush=True)

        # Estratégia 2: tentar ler o clipboard via page.evaluate (apenas se não headless)
        # Não implementar — clipboard não é confiável em headless e requer permissão

        print("[scraper] JSON de export não encontrado no DOM — sem maestrias", flush=True)
        return None

    except Exception as exc:
        print(f"[scraper] Export falhou (ignorando): {exc}", flush=True)
        return None
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05 | Tampering | JSON capturado do modal Questlog | mitigate | `json.loads()` com try/except — JSON malformado resulta em None (fallback gracioso). O Plan 01 aplica type guards adicionais no lado TypeScript |
| T-11-06 | Denial of Service | _try_export_specialization bloqueando por timeout longo | mitigate | `page.wait_for_timeout(1500)` limita a espera a 1.5s. O try/except garante que qualquer exceção retorna None sem bloquear o processo |
| T-11-07 | Information Disclosure | print() debug expondo stack trace no stderr | accept | Mensagens de print usam prefixo `[scraper]` e descrevem ação, não dados internos. O handler IPC (`electron/main/index.ts`) já separa stdout de stderr |
| T-11-08 | Tampering | Injeção via campo `mainHand`/`offHand` do JSON do Questlog | accept | Esses campos são armazenados como strings e nunca usados como path de arquivo ou parâmetro de sistema nesta fase — risco inexistente até Phase 12 usar os valores |
